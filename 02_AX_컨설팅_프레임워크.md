# AX 컨설팅 프레임워크 — AXEL Framework

> 📂 관련 문서: [AX란?](./00_AX란_무엇인가.md) · [브랜딩 전략](./01_AX전문가_퍼스널브랜딩_전략.md) · [실행 로드맵](./03_실행_로드맵.md)

> **AXEL = Assess → X-ray → Execute → Leverage**
> 기업의 AI 전환(AX)을 체계적으로 설계하고 실행하기 위한 4단계 방법론

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌──────────┐
│ Assess  │ →  │ X-ray   │ →  │ Execute │ →  │ Leverage │
│ (진단)   │    │ (설계)   │    │ (실행)   │    │ (확산)    │
│         │    │         │    │         │    │          │
│ 현재     │    │ 미래     │    │ 실행     │    │ 최적화    │
│ 파악     │    │ 설계     │    │ 구현     │    │ 확산      │
└─────────┘    └─────────┘    └─────────┘    └──────────┘
   2~3주          3~4주          8~12주         지속적
```

---

## Phase 1: Assess (진단) — 2~3주

> "현재 우리 조직은 어디에 있는가?"

### 핵심 활동
- **AX 성숙도 진단**: 5대 영역별 Level 0~5 측정
- **페인포인트 인터뷰**: 경영진·팀장·실무자 3계층 심층 면담
- **데이터 준비도 평가**: 데이터 품질·접근성·거버넌스 수준 점검
- **기술 환경 감사**: 기존 IT 인프라, 도구, 기술 스택 분석

### 📋 AX 성숙도 진단 체크리스트 (25개 항목)

> 각 항목을 0~5점으로 평가합니다. 총점 기준으로 조직의 AX 성숙도 단계를 판단합니다.

#### 전략 영역 (Strategy)

| # | 진단 질문 | 점수 (0~5) |
| --- | --- | --- |
| S1 | AI 관련 별도 예산이 배정되어 있는가? | ☐ |
| S2 | 경영진이 AI 전환의 필요성을 명확히 인식하고 있는가? | ☐ |
| S3 | AI 도입 목적이 구체적인 비즈니스 목표와 연결되어 있는가? | ☐ |
| S4 | AI 전환 로드맵(단기/중기/장기)이 문서화되어 있는가? | ☐ |
| S5 | AI 성과를 측정할 KPI가 정의되어 있는가? | ☐ |

#### 데이터 영역 (Data)

| # | 진단 질문 | 점수 (0~5) |
| --- | --- | --- |
| D1 | 핵심 업무 데이터가 디지털화·구조화되어 있는가? | ☐ |
| D2 | 데이터 품질 관리(중복/결측/정합성) 프로세스가 있는가? | ☐ |
| D3 | 데이터 접근 권한과 보안 정책이 수립되어 있는가? | ☐ |
| D4 | 부서 간 데이터 사일로가 해소되어 있는가? | ☐ |
| D5 | AI 학습에 활용할 수 있는 충분한 데이터가 축적되어 있는가? | ☐ |

#### 기술 영역 (Technology)

| # | 진단 질문 | 점수 (0~5) |
| --- | --- | --- |
| T1 | 직원들이 업무에 AI 도구(ChatGPT, Copilot 등)를 활용하고 있는가? | ☐ |
| T2 | AI 모델을 개발·운영할 수 있는 기술 인프라가 있는가? | ☐ |
| T3 | 기존 시스템(ERP, CRM 등)과 AI의 연동이 가능한 상태인가? | ☐ |
| T4 | AI 관련 보안(데이터 유출, 할루시네이션 대응) 체계가 있는가? | ☐ |
| T5 | 최신 AI 기술(에이전틱 AI, RAG 등) 도입을 검토한 적이 있는가? | ☐ |

#### 프로세스 영역 (Process)

| # | 진단 질문 | 점수 (0~5) |
| --- | --- | --- |
| P1 | 반복적인 업무 프로세스가 문서화되어 있는가? | ☐ |
| P2 | AI 기반으로 자동화된 워크플로우가 1개 이상 운영 중인가? | ☐ |
| P3 | 의사결정 프로세스에 데이터/AI 분석이 활용되고 있는가? | ☐ |
| P4 | 고객 접점(응대, 마케팅 등)에 AI가 활용되고 있는가? | ☐ |
| P5 | 업무 효율화를 위한 자동화 도구(RPA, n8n 등)를 사용 중인가? | ☐ |

#### 조직 영역 (Organization)

| # | 진단 질문 | 점수 (0~5) |
| --- | --- | --- |
| O1 | AI 관련 교육 프로그램이 운영되고 있는가? | ☐ |
| O2 | AI 전담 인력 또는 조직(CoE, TF 등)이 있는가? | ☐ |
| O3 | 직원들의 AI 활용에 대한 거부감이 낮은가? | ☐ |
| O4 | AI 도입에 대한 내부 커뮤니케이션/변화 관리가 진행 중인가? | ☐ |
| O5 | AI 거버넌스 위원회 또는 정책이 수립되어 있는가? | ☐ |

#### 점수 해석 기준

| 총점 | 성숙도 단계 | 의미 | 권장 액션 |
| --- | --- | --- | --- |
| 0~20 | Level 0 무관심 | AI 인식 부재 | 경영진 인식 전환 워크숍 |
| 21~40 | Level 1 실험 | 개인적 AI 사용 | 전사 AI 비전 수립 |
| 41~60 | Level 2 파일럿 | 부분적 프로젝트 | Quick Win 파일럿 확대 |
| 61~80 | Level 3 확산 | 다수 부서 도입 | 표준화 & 거버넌스 구축 |
| 81~100 | Level 4 내재화 | 전사적 AI 통합 | 최적화 & 자율화 추진 |
| 101~125 | Level 5 자율 운영 | AI 주도 운영 | 혁신 지속 & 업계 선도 |

### 산출물
- [ ] AX 성숙도 진단 보고서 (25개 항목 점수 + 5대 영역 레이더 차트)
- [ ] 핵심 페인포인트 맵 (3계층 인터뷰 종합)
- [ ] 데이터 준비도 스코어카드
- [ ] 현재 기술 환경 아키텍처 맵

## Phase 2: X-ray (설계) — 3~4주

> "어디로 가야 하며, 어떤 순서로 갈 것인가?"

### 핵심 활동
- **AI 비전 워크숍**: 경영진과 함께 AX 비전 및 목표 수립
- **유스케이스 발굴**: Impact × Feasibility 매트릭스로 우선순위 선정
- **로드맵 설계**: Quick Win → Scale → Transform 3단계 로드맵
- **ROI 모델링**: 예상 비용·효과·기간 산출

### 📋 유스케이스 우선순위 매트릭스

```
              높은 Impact (비즈니스 효과)
                       │
         ┌─────────────┼─────────────┐
         │             │             │
         │  Phase 2    │  Phase 1    │
         │ (전략적     │ (Quick Win)  │
         │  투자)      │  ★ 먼저!    │
         │             │             │
  낮은 ──┼─────────────┼─────────────┼── 높은
 구현    │             │             │  구현
 가능성  │  Phase 4    │  Phase 3    │  가능성
         │ (보류/관찰) │ (점진적     │
         │             │  개선)      │
         │             │             │
         └─────────────┼─────────────┘
                       │
              낮은 Impact
```

### 유스케이스 평가 템플릿

| 유스케이스 | Impact (1~5) | Feasibility (1~5) | 총점 | 우선순위 |
| --- | --- | --- | --- | --- |
| 예: 고객 문의 AI 자동 응답 | 4 | 5 | 20 | Phase 1 ★ |
| 예: 생산 품질 예측 분석 | 5 | 2 | 10 | Phase 2 |
| 예: 신규 사업 기회 발굴 AI | 3 | 1 | 3 | Phase 4 |

**Impact 평가 기준**: 비용 절감, 매출 증대, 고객 만족도, 의사결정 속도
**Feasibility 평가 기준**: 데이터 준비도, 기술 난이도, 조직 수용도, 소요 기간

### 3단계 로드맵 프레임

```
[Quick Win: 0~3개월]
  └ 즉시 효과를 볼 수 있는 1~3개 유스케이스
  └ 목적: 조직 내 AI 성공 경험 만들기
  └ 예: 회의록 자동 요약, FAQ 챗봇, 문서 분류

[Scale: 3~12개월]
  └ Quick Win의 성공을 바탕으로 확장
  └ 목적: 핵심 프로세스에 AI 내재화
  └ 예: 영업 예측, 마케팅 자동화, 품질 검사 AI

[Transform: 12~36개월]
  └ 비즈니스 모델 자체의 변화
  └ 목적: AI 네이티브 조직으로 전환
  └ 예: AI 기반 신규 서비스, 자율 운영 프로세스
```

### 산출물
- [ ] AX 비전 선언문 (1페이지)
- [ ] 유스케이스 우선순위 매트릭스 (최소 10개 후보)
- [ ] Quick Win / Scale / Transform 3단계 로드맵
- [ ] ROI 예측 모델 (투자 비용 vs 예상 효과)

## Phase 3: Execute (실행) — 8~12주

> "계획을 현실로 만든다"

### 핵심 활동
- **파일럿 프로젝트**: Quick Win 유스케이스 1~3개 선별 실행
- **기술 구현**: AI 에이전트, RAG 시스템, 워크플로우 자동화 구축
- **데이터 파이프라인**: 필요 데이터 수집·정제·구조화
- **변화 관리**: 직원 교육, 커뮤니케이션 계획 실행

### 📋 파일럿 프로젝트 설계 템플릿

각 파일럿 프로젝트는 아래 항목을 사전에 정의한 후 착수합니다.

```
┌─────────────────────────────────────────────────────┐
│              파일럿 프로젝트 기획서                     │
├──────────┬──────────────────────────────────────────┤
│ 프로젝트명 │                                          │
├──────────┼──────────────────────────────────────────┤
│ 목적      │ 어떤 문제를 해결하는가?                     │
├──────────┼──────────────────────────────────────────┤
│ 범위      │ 어떤 부서/프로세스에 적용하는가?              │
├──────────┼──────────────────────────────────────────┤
│ 기술      │ 어떤 AI 기술을 사용하는가?                   │
│          │ (LLM / RAG / 에이전트 / 자동화 등)           │
├──────────┼──────────────────────────────────────────┤
│ 데이터    │ 어떤 데이터가 필요한가? 현재 확보 상태는?       │
├──────────┼──────────────────────────────────────────┤
│ 성공 지표  │ 무엇으로 성공을 측정하는가? (정량 KPI)         │
├──────────┼──────────────────────────────────────────┤
│ 일정      │ 파일럿 기간은? (주 단위 마일스톤)              │
├──────────┼──────────────────────────────────────────┤
│ 담당자     │ 프로젝트 오너 / 기술 담당 / 현업 담당          │
├──────────┼──────────────────────────────────────────┤
│ 리스크     │ 예상되는 위험과 대응 방안은?                   │
├──────────┼──────────────────────────────────────────┤
│ 비용      │ 예상 투자 비용 (인건비, 라이선스, 인프라)       │
└──────────┴──────────────────────────────────────────┘
```

### 변화 관리 커뮤니케이션 플랜

| 대상 | 메시지 | 채널 | 빈도 |
| --- | --- | --- | --- |
| 경영진 | 진행 상황, ROI 데이터, 리스크 | 주간 보고서 | 주 1회 |
| 팀장 | 파일럿 결과, 팀 적용 방법 | 워크숍, 데모 | 격주 1회 |
| 전 직원 | AI 활용 사례, 교육 안내 | 뉴스레터, 인트라넷 | 월 1회 |

### 산출물
- [ ] 파일럿 프로젝트 실행 보고서 (Before/After 비교)
- [ ] AI 시스템 구축 결과물 (데모 가능 상태)
- [ ] 교육 프로그램 커리큘럼 & 실행 결과
- [ ] 변화 관리 커뮤니케이션 이력

## Phase 4: Leverage (확산 & 최적화) — 지속적

> "성공을 조직 전체로 확산한다"

### 핵심 활동
- **성과 측정**: 파일럿 결과 정량/정성 평가
- **확산 전략**: 성공 사례를 다른 부서/프로세스로 확대
- **거버넌스 구축**: AI 윤리, 보안, 품질 관리 체계 수립
- **지속적 최적화**: 피드백 루프를 통한 성능·프로세스 개선

### 📋 AI 거버넌스 체크리스트

#### 정책 수립
- [ ] AI 사용 정책(Acceptable Use Policy) 수립 및 전 직원 공유
- [ ] AI 활용 가능 업무 / 금지 업무 경계 정의
- [ ] 외부 AI 서비스 사용 시 데이터 전송 범위 규정

#### 데이터 & 보안
- [ ] AI에 입력 가능한 데이터 등급 분류 (공개/내부/기밀)
- [ ] 개인정보 처리 시 비식별화 프로세스 적용
- [ ] 데이터 유출 방지(DLP) 체계와 AI 연동 확인
- [ ] AI 학습 데이터의 저작권·라이선스 검증

#### 품질 & 신뢰성
- [ ] AI 산출물의 할루시네이션 검증 프로토콜 수립
- [ ] 주요 의사결정에 Human-in-the-loop 규정 적용
- [ ] AI 모델 성능 모니터링 및 정기 평가 주기 설정
- [ ] AI 오류/사고 발생 시 대응 프로세스(에스컬레이션) 문서화

#### 윤리 & 공정성
- [ ] AI 편향(Bias) 모니터링 및 교정 절차 수립
- [ ] AI 의사결정의 설명 가능성(Explainability) 확보
- [ ] 이해관계자(고객, 직원)에 AI 활용 사실 투명하게 공개

#### 감사 & 거버넌스 체계
- [ ] AI 거버넌스 위원회(또는 담당자) 지정
- [ ] 정기 AI 감사(Audit) 일정 수립 (분기/반기)
- [ ] AI 관련 법규 변경 모니터링 담당 지정

### 성과 보고 템플릿

| 항목 | Before (도입 전) | After (도입 후) | 개선율 |
| --- | --- | --- | --- |
| 처리 시간 | 예: 2시간/건 | 예: 15분/건 | 87.5% ↓ |
| 오류율 | 예: 5% | 예: 1.2% | 76% ↓ |
| 비용 | 예: 월 500만원 | 예: 월 120만원 | 76% ↓ |
| 고객 만족도 | 예: 3.2/5 | 예: 4.5/5 | 40.6% ↑ |

### 산출물
- [ ] 파일럿 성과 보고서 (Before/After 비교, ROI 검증)
- [ ] 확산 로드맵 (2차, 3차 적용 대상 부서/프로세스)
- [ ] AI 거버넌스 정책 문서
- [ ] 지속적 모니터링 대시보드 설계

## 부록: AXEL 프레임워크 한눈에 보기

| Phase | 목적 | 기간 | 핵심 질문 | 핵심 산출물 |
| --- | --- | --- | --- | --- |
| **Assess** | 진단 | 2~3주 | "지금 어디에 있는가?" | 성숙도 보고서 |
| **X-ray** | 설계 | 3~4주 | "어디로 갈 것인가?" | 로드맵, ROI 모델 |
| **Execute** | 실행 | 8~12주 | "어떻게 만들 것인가?" | 파일럿 결과물 |
| **Leverage** | 확산 | 지속적 | "어떻게 퍼뜨릴 것인가?" | 거버넌스, 확산 계획 |

> 이 프레임워크는 AX 컨설팅의 **기본 골격**이며, 기업의 규모·산업·성숙도에 따라 유연하게 조정됩니다.
> 각 Phase의 기간은 중소기업(50~200명) 기준이며, 대기업은 1.5~2배 소요될 수 있습니다.
