## Dify가 뭔가요?

> **"레고 블록처럼 AI 앱을 조립하는 플랫폼"**

Dify는 ChatGPT 같은 AI 모델을 활용하여 나만의 AI 앱(챗봇, 워크플로우, 에이전트)을 **드래그 & 드롭**으로 만들 수 있는 오픈소스 플랫폼입니다.

```
일반 ChatGPT 사용           Dify로 만든 AI
─────────────────          ─────────────────
• 범용 지식으로 답변          • 우리 회사 문서 기반 답변
• 대화 끝나면 잊어버림        • 지식 베이스에 영구 저장
• 직접 사이트에서 사용        • 웹/Slack/API로 어디든 배포
• 커스터마이즈 어려움         • 프롬프트, 모델, 흐름 자유 설계
```

## 시작하기: 2가지 설치 방법

### 방법 A: 클라우드 (가장 빠름 — 3분)

1. https://cloud.dify.ai 접속
2. 구글/깃허브 계정으로 가입
3. 바로 사용 시작!

> 💡 무료 플랜: 월 200회 메시지, 학습용으로 충분

### 방법 B: 로컬 설치 — Docker (기업용, 보안 우선)

```bash
# 1. 저장소 클론
git clone https://github.com/langgenius/dify.git
cd dify/docker

# 2. 환경 파일 복사
cp .env.example .env

# 3. Docker 실행
docker compose up -d

# 4. 브라우저에서 접속
# http://localhost/install
```

> 💡 로컬 설치 시 모든 데이터가 내 서버에 머물러 **보안에 유리** → 기업 파일럿에 적합

# 🎓 프로젝트 1: 사내 FAQ 챗봇 (30분)

> **상황**: 신규 입사자들이 "연차 신청은 어떻게 하나요?", "출장비 정산은?" 같은 질문을 매번 총무팀에 한다. 이걸 AI가 대신 답변하게 만들자.

## Step 1: 지식 베이스 만들기

```
[지식(Knowledge)] 탭 → [지식 생성(Create Knowledge)] 클릭
```

1. **이름**: "사내 규정집"
2. **데이터 업로드**: 회사 규정 PDF, 인사 가이드 문서 등 업로드
3. **청킹 설정**: "자동" 선택 (Dify가 알아서 적절히 쪼개줌)
4. **저장 & 처리** 클릭 → 인덱싱 완료까지 1~5분 대기

```
업로드 가능한 파일 형식:
├── PDF (가장 많이 사용)
├── TXT, Markdown
├── DOCX, XLSX
├── Notion 페이지 연동
└── 웹사이트 URL 크롤링
```

## Step 2: 챗봇 앱 만들기

```
[스튜디오(Studio)] → [빈 상태로 시작] → [챗봇] 선택
```

1. **앱 이름**: "HR 도우미"
2. **모델 선택**: GPT-4o-mini (빠르고 저렴)
3. **시스템 프롬프트** 작성:

```
당신은 [회사명]의 인사/총무 전문 AI 어시스턴트입니다.

규칙:
1. 반드시 제공된 지식 베이스 내 정보만을 기반으로 답변하세요.
2. 지식 베이스에 없는 내용은 "해당 정보는 확인되지 않습니다.
   총무팀(내선 1234)에 문의해 주세요."라고 안내하세요.
3. 답변은 친절하고 간결하게, 핵심을 먼저 말하세요.
4. 관련 규정의 조항 번호를 함께 안내해 주세요.
```

4. **컨텍스트(Context)** → [추가] → "사내 규정집" 지식 베이스 선택

## Step 3: 테스트 & 배포

1. 오른쪽 **미리보기(Preview)** 패널에서 질문 테스트
   - "연차 신청 절차 알려줘"
   - "출장비 정산 기한이 어떻게 되나요?"
   - "점심시간은 몇 시부터?"

2. 답변 품질 확인 후 **발행(Publish)** 클릭

3. 배포 옵션 선택:

```
배포 방법:
├── 🌐 웹앱 링크: URL 공유만으로 바로 사용
├── 📋 임베드 코드: 사내 인트라넷에 삽입
└── 🔌 API: 카카오톡/Slack 등과 연동
```

### ✅ 완성! 30분 만에 사내 FAQ 챗봇 완성

# 🎓 프로젝트 2: 고객 문의 자동 분류기 (워크플로우)

> **상황**: 고객 문의 이메일이 하루 50통 들어온다. AI가 카테고리를 분류하고 담당 부서에 자동 전달하게 만들자.

## 워크플로우 개념

```
[시작] → [LLM 분류] → [조건 분기] → [결과 출력]
                          ├── 제품 문의 → 영업팀
                          ├── 기술 지원 → 기술팀
                          ├── 불만/클레임 → CS팀
                          └── 기타 → 총무팀
```

## Step 1: 워크플로우 앱 생성

```
[스튜디오] → [빈 상태로 시작] → [워크플로우(Workflow)] 선택
```

## Step 2: 노드 구성

### 노드 ①: 시작 (Start)
- 입력 변수 추가: `customer_email` (텍스트, 여러 줄)

### 노드 ②: LLM (분류)
- 모델: GPT-4o-mini
- 프롬프트:

```
다음 고객 이메일을 읽고, 아래 4개 카테고리 중 하나로 분류하세요.
반드시 카테고리 이름만 출력하세요.

카테고리:
- 제품문의
- 기술지원
- 불만클레임
- 기타

이메일 내용:
{{customer_email}}
```

### 노드 ③: IF/ELSE (조건 분기)
```
조건 1: LLM 출력 = "제품문의"   → 분기 A
조건 2: LLM 출력 = "기술지원"   → 분기 B
조건 3: LLM 출력 = "불만클레임" → 분기 C
기타                           → 분기 D
```

### 노드 ④: 각 분기별 응답
- 분기 A → "영업팀(sales@company.com)으로 전달되었습니다"
- 분기 B → "기술지원팀(support@company.com)으로 전달되었습니다"
- 분기 C → "CS팀(cs@company.com)으로 긴급 전달되었습니다"
- 분기 D → "총무팀으로 전달되었습니다"

## Step 3: 테스트

디버그 패널에서 테스트 이메일 입력:

```
"안녕하세요. 지난주에 구매한 노트북에서 화면이 깜빡이는 
현상이 계속됩니다. A/S 접수하려면 어떻게 해야 하나요?"
```

→ 기대 결과: "기술지원" 분류 → "기술지원팀으로 전달되었습니다"

# 🎓 프로젝트 3: AI 에이전트 — 도구를 쓰는 챗봇

> **상황**: 챗봇이 단순 답변을 넘어, 인터넷 검색도 하고, 날씨도 확인하고, 계산도 할 수 있게 만들자.

## 에이전트 vs 챗봇 차이

```
일반 챗봇:  질문 → AI가 알고 있는 것만 답변
에이전트:   질문 → AI가 "어떤 도구를 써야 할지" 판단 → 도구 실행 → 결과로 답변
```

## Step 1: 에이전트 앱 생성

```
[스튜디오] → [빈 상태로 시작] → [에이전트(Agent)] 선택
```

## Step 2: 도구(Tool) 추가

Dify에서 제공하는 내장 도구들:

| 도구 | 기능 | 활용 예시 |
| --- | --- | --- |
| **웹 검색** | 실시간 인터넷 검색 | "최신 AI 트렌드 알려줘" |
| **위키피디아** | 백과사전 검색 | "RAG가 뭔지 설명해줘" |
| **날씨** | 날씨 정보 조회 | "서울 내일 날씨는?" |
| **계산기** | 수학 계산 | "1,200만원의 부가세 포함 가격은?" |
| **DALL-E** | 이미지 생성 | "로고 이미지 만들어줘" |
| **웹 크롤링** | 웹페이지 내용 읽기 | "이 URL 요약해줘" |

도구 추가 방법:
```
에이전트 설정 → [도구(Tools)] → [추가] → 원하는 도구 선택
```

## Step 3: 에이전트 프롬프트

```
당신은 비즈니스 리서치 어시스턴트입니다.

사용자의 질문에 답변할 때:
1. 먼저 지식 베이스에서 관련 정보를 검색합니다.
2. 지식 베이스에 없으면 웹 검색 도구를 사용합니다.
3. 숫자 계산이 필요하면 계산기 도구를 사용합니다.
4. 답변에는 항상 출처를 표기합니다.
```

# 🎓 프로젝트 4: 비즈니스 보고서 생성기 (고급 워크플로우)

> **상황**: 키워드만 입력하면 AI가 자동으로 시장 조사 → 분석 → 보고서 초안을 작성

## 워크플로우 설계

```
[키워드 입력]
     ↓
[웹 검색 노드] ← "키워드로 최신 기사 5건 검색"
     ↓
[LLM 노드 1] ← "검색 결과 요약 & 핵심 포인트 추출"
     ↓
[지식 검색 노드] ← "내부 문서에서 관련 데이터 검색"
     ↓
[LLM 노드 2] ← "외부 + 내부 정보를 종합하여 보고서 초안 작성"
     ↓
[최종 출력]
```

### 최종 출력 프롬프트 예시:

```
다음 정보를 바탕으로 비즈니스 보고서 초안을 작성하세요.

형식:
## 1. 개요
## 2. 시장 현황
## 3. 핵심 인사이트 (3가지)
## 4. 기회와 위협
## 5. 제안 사항

웹 검색 결과: {{web_search_results}}
내부 문서 검색 결과: {{knowledge_results}}
```

## Dify 핵심 용어 정리

| 용어 | 설명 | 비유 |
| --- | --- | --- |
| **스튜디오 (Studio)** | 앱을 만드는 작업 공간 | 공장 |
| **지식 (Knowledge)** | 문서를 AI가 검색할 수 있게 저장 | 도서관 |
| **챗봇 (Chatbot)** | 대화형 AI 앱 | 안내 데스크 |
| **워크플로우 (Workflow)** | 노드를 연결한 자동화 흐름 | 조립 라인 |
| **에이전트 (Agent)** | 도구를 스스로 선택하는 AI | 만능 비서 |
| **노드 (Node)** | 워크플로우의 각 단계 (블록) | 레고 블록 |
| **프롬프트 (Prompt)** | AI에게 주는 지시사항 | 업무 지시서 |
| **RAG** | 문서 검색 후 AI가 답변하는 방식 | 참고서 보고 답하기 |
| **컨텍스트 (Context)** | 지식 베이스 연결 설정 | 참고 자료 지정 |
| **발행 (Publish)** | 완성된 앱을 사용 가능 상태로 배포 | 제품 출시 |

## AX 컨설턴트를 위한 Dify 활용 치트시트

### 고객 미팅 때 30분 라이브 데모 시나리오

```
[0분] "귀사의 업무 매뉴얼 PDF 파일 하나만 주세요"
       → 고객이 파일 제공

[5분] 지식 베이스 생성 & 파일 업로드
       → 인덱싱 진행 중 잠깐 대화

[10분] 챗봇 앱 생성 & 프롬프트 설정

[15분] 지식 베이스 연결

[20분] 테스트 — 고객이 직접 질문 입력
       → "이 문서에 있는 내용으로 AI가 바로 답변합니다"

[25분] 고객 반응: "오... 이게 이렇게 쉽게 되나요?"

[30분] "이것이 Phase 1입니다. 여기서 워크플로우와 
        에이전트를 추가하면 더 강력해집니다."
       → AXEL 프레임워크 소개로 자연스럽게 연결
```

### 파일럿 프로젝트별 예상 일정

| 프로젝트 | Dify 기능 | 소요 시간 | 난이도 |
| --- | --- | --- | --- |
| 사내 FAQ 챗봇 | 챗봇 + 지식 | 반나절 | ⭐ |
| 문의 분류 자동화 | 워크플로우 | 1~2일 | ⭐⭐ |
