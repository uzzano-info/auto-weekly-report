# Auto-Weekly Report Swarm (주간 보고서 자동화) 고도화 튜토리얼

이전 튜토리얼에서 구축한 기본 파이프라인을 실무 및 포트폴리오 수준으로 끌어올리기 위한 **심화 튜토리얼**입니다. 총 4단계에 걸쳐 코드를 업그레이드합니다.

---

## 📅 Step 1: 동적 날짜 적용 및 파일명 자동화

기존 코드는 항상 `Weekly_Report.md`로 파일을 덮어쓰고, AI가 이번 주가 언제인지 정확히 알지 못했습니다. 파이썬의 `datetime`을 활용해 이를 개선합니다.

**1. `main.py` 수정**
날짜가 포함된 파일명으로 저장하도록 `main.py`를 수정합니다.

```python
# main.py
from dotenv import load_dotenv
load_dotenv()

from agents import build_graph
from datetime import datetime

def main():
    print("🚀 Auto-Weekly Report Swarm 시작!\n")
    graph = build_graph()
    
    # 오늘 날짜 추출
    today_str = datetime.now().strftime("%Y-%m-%d")
    
    # 초기 상태에 날짜 정보 주입 (선택 사항: agents.py의 ReportState 구조 변경 필요)
    initial_state = {"raw_data": "", "analyzed_data": "", "final_report": ""}
    
    result = graph.invoke(initial_state)
    report_content = result["final_report"]
    
    # 동적 파일명 생성
    output_filename = f"Weekly_Report_{today_str}.md"
    
    with open(output_filename, "w", encoding="utf-8") as f:
        f.write(report_content)
        
    print(f"\n✅ 완료! 최종 보고서가 '{output_filename}' 파일로 저장되었습니다.")

if __name__ == "__main__":
    main()
```

**2. `agents.py`의 프롬프트에 날짜 반영 (참고)**
`analyzer_node`나 `writer_node`의 프롬프트에 오늘 날짜 정보를 시스템 메시지로 넘겨주면, LLM이 더 정확한 시제(과거/미래)를 사용하여 리포트를 작성할 수 있습니다.

## 🖥️ Step 2: Streamlit을 활용한 Web UI 구축

터미널 실행을 넘어, 누구나 쉽게 버튼을 눌러 사용할 수 있는 웹 대시보드를 만듭니다. 포트폴리오 시각화에 매우 효과적입니다.

**1. Streamlit 설치**
터미널에서 아래 명령어로 패키지를 설치합니다.
```bash
pip install streamlit
```

**2. `app.py` 생성 및 작성**
프로젝트 폴더 안에 `app.py`를 만들고 아래 코드를 넣습니다.

```python
# app.py
import streamlit as st
import os
from dotenv import load_dotenv

load_dotenv()

from agents import build_graph

st.set_page_config(page_title="주간 보고서 자동화", page_icon="📝")

st.title("📝 오토 위클리 리포트 생성기")
st.write("버튼을 누르면 로컬 폴더의 메모와 Git 로그를 취합하여 주간 보고서를 작성합니다.")

if st.button("보고서 생성 시작", type="primary"):
    with st.spinner("AI가 데이터를 수집하고 분석 중입니다... 잠시만 기다려주세요."):
        # LangGraph 실행
        graph = build_graph()
        initial_state = {"raw_data": "", "analyzed_data": "", "final_report": ""}
        result = graph.invoke(initial_state)
        
        report = result["final_report"]
        
    st.success("✅ 보고서 생성 완료!")
    
    # 결과 화면 출력
    st.markdown("### 📊 생성된 보고서")
    st.info(report)
    
    # 다운로드 버튼
    st.download_button(
        label="📥 마크다운 파일 다운로드",
        data=report,
        file_name="Weekly_Report.md",
        mime="text/markdown"
    )
```

**3. 앱 실행하기**
가상환경이 켜진 터미널에서 아래 명령어로 웹 앱을 실행합니다.
```bash
streamlit run app.py
```
자동으로 브라우저가 열리며 훌륭한 UI의 웹 애플리케이션을 확인할 수 있습니다.

## 🧠 Step 3: LangGraph 메모리(Checkpointer) 도입

한 번 작성된 보고서에서 그치지 않고, "이슈 부분을 조금 더 부드럽게 고쳐줘" 같은 추가 피드백을 반영할 수 있도록 에이전트에 **기억(Memory)**을 부여합니다.

**1. `agents.py`에 MemorySaver 추가**
LangGraph의 내장 메모리를 활용하여 그래프를 컴파일합니다.

```python
from langgraph.checkpoint.memory import MemorySaver

def build_graph():
    workflow = StateGraph(ReportState)
    
    # (노드 및 엣지 추가 부분은 기존과 동일)
    
    # 메모리 객체 생성
    memory = MemorySaver()
    
    # 컴파일 시 checkpointer 전달
    return workflow.compile(checkpointer=memory)
```

**2. 스레드(Thread) ID를 사용한 호출**
그래프를 호출할 때 고유한 `thread_id`를 설정하여 넘겨주면, 그래프가 해당 ID를 가진 대화의 이전 상태를 기억하게 됩니다.

```python
config = {"configurable": {"thread_id": "user_123"}}
result = graph.invoke(initial_state, config=config)
```

> ⚠️ **주의사항 (최신 LangGraph 패턴)**: 
> 단순한 텍스트 덮어쓰기(`TypedDict`) 구조에서는 상태를 매번 초기화하면 기억이 날아갑니다. 진정한 피드백 루프를 만들려면 아래처럼 `agents.py`의 `ReportState`에 사용자와의 대화 내역을 누적하는 `messages` 필드를 추가해야 합니다.
> ```python
> from typing import Annotated, TypedDict
> from langgraph.graph.message import add_messages
> 
> class ReportState(TypedDict):
>     messages: Annotated[list, add_messages] # 대화 내역 누적
>     raw_data: str       
>     analyzed_data: str  
>     final_report: str   
> ```
> 이후 Streamlit의 채팅 UI(`st.chat_message`)와 결합하면 완벽한 휴먼 인 더 루프(Human-in-the-loop)가 완성됩니다.

## 🔗 Step 4: 외부 SaaS 연동 (Tool Expansion)

로컬 텍스트 파일 대신, 실제 협업 툴의 데이터를 읽어오도록 `tools.py`를 확장합니다. 

**아이디어 및 확장 예시:**
* **Slack API 연동:** 이번 주 특정 업무 채널에 올라온 주요 메시지들을 크롤링하는 도구. (`slack_sdk` 라이브러리 사용)
* **Jira / Linear 연동:** 본인에게 할당된 티켓 중 이번 주에 'Done' 혹은 'In Progress'로 상태가 변경된 이슈 목록 가져오기.
* **Notion API:** 노션에 작성한 일일 회의록 데이터 파싱.

`tools.py`에 위 API를 호출하는 파이썬 함수를 만들고 `@tool` 데코레이터를 붙이기만 하면, 수집가(Collector) 노드가 해당 데이터를 가져와 훨씬 더 입체적이고 구체적인 리포트를 작성하게 됩니다.
